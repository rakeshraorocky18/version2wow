# WOW - World of Weddings | Team Setup Guide

## Prerequisites

Install the following on your workstation before running the project.

### 1. Node.js (Required)

Download and install **Node.js LTS (v20+)** from:
- https://nodejs.org/en/download

Or via terminal (Windows with winget):
```powershell
winget install OpenJS.NodeJS.LTS
```

After installation, **restart your terminal** and verify:
```powershell
node --version    # Should show v20.x.x or higher
npm --version     # Should show 10.x.x or higher
```

> **Important:** If `node` or `npm` is not recognized after install, add `C:\Program Files\nodejs` to your system PATH manually:
> Settings → System → About → Advanced system settings → Environment Variables → Edit PATH → Add `C:\Program Files\nodejs`

### 2. Git (Recommended)

```powershell
winget install Git.Git
```

### 3. VS Code (Recommended)

```powershell
winget install Microsoft.VisualStudioCode
```

Recommended VS Code extensions:
- ESLint
- Prettier
- Tailwind CSS IntelliSense
- Thunder Client (API testing)

### 4. Docker (Optional — for PostgreSQL/MongoDB/Redis)

Only needed when you want to switch from SQLite to production databases.

```powershell
winget install Docker.DockerDesktop
```

## Project Setup

The project is expected at: `C:\Projects\wow-world-of-weddings`

### Step 1: Install Dependencies

Open a terminal and run:

```powershell
cd C:\Projects\wow-world-of-weddings\backend
npm install

cd C:\Projects\wow-world-of-weddings\frontend
npm install
```

### Step 2: Environment Configuration

A `.env` file should already exist at the project root (`C:\Projects\wow-world-of-weddings\.env`).
If missing, copy from the example:

```powershell
cd C:\Projects\wow-world-of-weddings
copy .env.example .env
```

Default `.env` values work out of the box with SQLite (no external databases needed).

### Step 3: Build the Backend

```powershell
cd C:\Projects\wow-world-of-weddings\backend
npx nest build
```

This compiles TypeScript to `dist/` folder. Must succeed with 0 errors before running.

### Step 4: Start the Backend

```powershell
cd C:\Projects\wow-world-of-weddings\backend
npx nest start
```

You should see:
```
🎉 WOW API running on http://localhost:3000
📖 Swagger docs at http://localhost:3000/api/docs
```

### Step 5: Start the Frontend (new terminal)

Open a **second terminal**:

```powershell
cd C:\Projects\wow-world-of-weddings\frontend
npx vite
```

You should see:
```
VITE ready
➜ Local: http://localhost:5173/
```

### Step 6: Access the Application

| Service          | URL                              |
|------------------|----------------------------------|
| Frontend         | http://localhost:5173             |
| Backend API      | http://localhost:3000/api         |
| Swagger API Docs | http://localhost:3000/api/docs    |
| WebSocket (Chat) | ws://localhost:3000/chat          |

## Database

### Current: SQLite (Development)

The backend uses **SQLite** (`wow_dev.db` file in the backend folder) for zero-setup development. No external database installation needed. The DB file is auto-created on first run.

To reset the database, simply delete `wow_dev.db` and restart the server.

### Production: PostgreSQL + MongoDB + Redis

When ready for production, update `app.module.ts` to use PostgreSQL and start databases via Docker:

```powershell
cd C:\Projects\wow-world-of-weddings
docker-compose -f docker/docker-compose.yml up postgres mongodb redis -d
```

## Troubleshooting

| Issue | Solution |
|-------|---------|
| `node` not recognized | Restart terminal; add `C:\Program Files\nodejs` to system PATH |
| `npx nest` not recognized | Run `npm install` in the backend folder first |
| Port 3000 in use | Kill the process: `Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess \| Stop-Process` |
| Port 5173 in use | Kill the process: `Get-Process -Id (Get-NetTCPConnection -LocalPort 5173).OwningProcess \| Stop-Process` |
| Build errors | Run `npm install` again, then `npx nest build` |
| SQLite DB corrupt | Delete `backend\wow_dev.db` and restart |
| bcrypt install fails | Install Visual Studio Build Tools: `npm install --global windows-build-tools` |

## What to Copy

When transferring to another workstation, copy the **entire** `C:\Projects\wow-world-of-weddings` folder **excluding**:

- `node_modules\` (will be regenerated by `npm install`)
- `dist\` (will be regenerated by `npx nest build`)
- `wow_dev.db` (will be auto-created on first run)

Quickest way:
```powershell
# On source machine — create a clean archive
cd C:\Projects
tar -czf wow-project.tar.gz --exclude="node_modules" --exclude="dist" --exclude="*.db" wow-world-of-weddings

# On target machine — extract and set up
cd C:\Projects
tar -xzf wow-project.tar.gz
cd wow-world-of-weddings\backend && npm install
cd ..\frontend && npm install
```

Or use Git (recommended for ongoing collaboration):
```powershell
cd C:\Projects\wow-world-of-weddings
git init && git add -A && git commit -m "Initial WOW project"
# Push to your team's Git remote
```
